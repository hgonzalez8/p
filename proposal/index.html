Summary

In this project, we plan to extend our path tracer to support volumetric scattering and render these effects using Bidirectional Path Tracing. To implement volumetric scattering we will use model angular scattering and Beer-Lambert's law to calculate light distribution and absorption caused by particles, allowing us to model light rays through participating media such as smoke, fog, and haze, where light can scatter before reaching a surface - rendering realistic effects. Our project will explore how BDPT captures the volumetric lighting effects and compare its performance and visual quality against the path tracing approach implemented in homework 3.

Problem Description

So far in class, we've worked with surface-based path tracing methods that use Monte Carlo estimation to model light transport through a scene. While this approach excels for rendering illumination on objects or surfaces, it cannot accurately capture volumetric effects such as beams from stagelights. These effects arise because light not only interacts with surfaces, but also with the participating medium in the space between them, where particles in the real-world scatter and absorb light before reaching a surface.

This is an important limitation because many real-world phenomena depend on volumetric light transport, and the standard surface-based path tracing method would incorrectly render these scenes. To render these effects, we need to extend the surface-only formulation so rays may undergo scattering events within a volume, which makes our modeling task much more complex. In particular, we need to determine how to sample scattering events in space and also account for any absorption or attenuation through a medium - doing all of this efficiently so rendering these effects is practical.

To address this problem, we plan to extend the path tracer implemented in Homework 3 to support volumetric scattering and implement Bidirectional Path Tracing (BDPT) as our main rendering algorithm. Unlike standard path tracing which builds paths only from the camera, BDPT constructs 2-way paths from both the camera and light source and connects them to form paths. We will also change our sampling method to consider the intersection of light rays with particles and how this can affect light distribution within our scenes. At minimum, we plan to implement volumetric scattering and run evaluations on image quality, convergence, and runtime. If we are ahead of schedule, we hope to incorporate and rework methods from lectures such as russian roulette or adaptive sampling to be compatible with BDPT.

Goals and Deliverables

Ultimately, we plan to deliver a complete implementation and an in-depth explanation of a bidirectional path tracing algorithm, and compare it against the global illumination function from homework 3 across various scenes containing light scattering objects. In addition to standard surface light transport, we plan to investigate scenes with participating media, where light scatters in a medium before reaching a surface. This will allow us to study how BDPT handles complex lighting effects like volumetric scattering in fog-like, smoke-like or illumination environments (e.g. stagelights). We will evaluate the effects of varying the samples per pixel, number of light rays, bounce depth and accumulations, and other various parameters used to render scenes from the assignment. We also plan to discuss whether techniques like Russian Roulette or adaptive sampling can improve the rendering times without compromising visual quality. Our evaluation will focus on visual realism (capturing real-world interaction between light and atmospheric particles), convergence behavior, rendering times, and memory usage across different scenes.

For this project, we plan to answer whether BDPT provides advantages over standard path tracing for complex scenes. We also want to answer the question of how to keep BDPT useful for real-time rendering applications while maintaining its intended effects on most scenes. If time permits, we hope to build a modified interactive GUI that supports our BDPT renderer. Ideally, we would like to keep most of the implemented features from homework 3, but we vision this modified version of the GUI will path trace in close to real time and allow the user to move and zoom the camera.
